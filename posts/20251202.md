### 思考過程

1. 直覺暴力解：三個指標，每個指標往後（index大的方向）搜尋。
2. 例如 nums = [5, 1, 4, 2, 3]->[5,1,4], [5,1,2] ... -> [1,4,2], [1,4,3],**[1,2,3]**，找到了！
3. 但這樣太慢，時間複雜度是 $O(n^3)$
4. 如果直接找最小(First) 跟第二小(Second) 的呢？
5. 遍歷每個數字，每個數字都先跟 First 再跟 Second 比較，若較小就替換掉
6. 因為順序的關係，若找到一個比 First 跟 Second 大的數字，那就 **找到了**！
7. GPT 說這就是貪婪法(?)

### 複雜度分析

- **時間複雜度**：$O(n)$，每個元素只處理一次。
- **空間複雜度**：$O(1)$，只儲存固定數量的變數。

## 程式碼（Python 解法）

```python
class Solution:
    def increasingTriplet(self, nums: List[int]) -> bool:
        # 儲存最小跟第二小的
        first = second = float('inf')

        for num in nums:
            # 先找個數字當最小的，存在 First
            if num <= first:
                first = num
            # 如果比 First 大，又比 Second 小，那就存為 Second
            elif num <= second:
                second = num
            # 如果比 First 跟 Second 都大，那就找到了！
            else:
                return True
        return False
```

## 小結 & 反思

- 直覺的暴力破解法通常都會遇到 TLE (Time Limit Exceeded)
```python
for i in range(n):
    for j in range(i+1, n):
        for k in range(j+1, n):
            if nums[i] < nums[j] and nums[j] < nums[k]:
                return True
return False

```
- 何時可以使用貪婪法？

  1. 只要求找到可行解（是否存在），不需最優解
     - Example: 是否能跳到終點、是否存在遞增三元組

  2. 當前最佳選擇不會影響未來成功機會（單調性）

  3. 做越多、越小、越便宜的選擇 → 之後一定更有利

  4. 每次決策不需要回頭修改（不可逆性）

  5. 做了選擇就不會後悔，不需回溯補救

  6. 局部最佳一定能導向全域最佳（問題結構可保證）

  7. 有可證明的最優子結構，或題目本身具有明確的貪婪策略
